멋진 테트리스 게임
제작 전략 가이드

단순히 블록을 떨어뜨리는 게임이 아니라, 구조가 탄탄하고 디자인이 세련되며 점수·레벨·효과음까지 갖춘 완성형 테트리스를 만드는 방법을 단계별로 배웁니다.

목표 제작 전략 핵심 로직 파일 구조 예제 코드 디자인 고급 기능 완성 체크
1. 프로젝트 목표

우리가 만들 테트리스

초보자는 먼저 기본 동작을 완성하고, 그다음 디자인과 기능을 추가해야 합니다. 처음부터 모든 기능을 한꺼번에 넣으면 오류를 찾기 어려워집니다.

A

기본 게임

블록 생성, 이동, 회전, 충돌, 줄 삭제, 게임 오버를 구현합니다.

B

재미 요소

점수, 레벨, 속도 증가, 다음 블록, 일시정지 기능을 넣습니다.

C

멋진 연출

네온 디자인, 애니메이션, 효과음, 반응형 화면을 적용합니다.

2. 전체 제작 전략

작은 기능부터 차례로 완성하기

게임판부터 만든다

가로 10칸, 세로 20칸의 2차원 배열을 만듭니다. 빈칸은 0, 블록이 있는 칸은 숫자로 저장합니다.

테트로미노 7종을 정의한다

I, J, L, O, S, T, Z 블록을 작은 2차원 배열로 만들어 관리합니다.

키보드 조작을 연결한다

왼쪽·오른쪽 이동, 아래로 내리기, 위쪽 키로 회전, 스페이스바로 즉시 낙하를 구현합니다.

충돌 판정을 만든다

벽, 바닥, 이미 쌓인 블록과 겹치지 않는지 검사합니다.

줄 삭제와 점수를 넣는다

가득 찬 줄을 제거하고 위의 줄을 아래로 내립니다. 여러 줄을 한 번에 지우면 보너스를 줍니다.

마지막에 디자인을 강화한다

게임이 정상 작동한 뒤 네온 색상, 그림자, 파티클, 사운드, 모바일 버튼을 추가합니다.

3. 핵심 게임 로직

테트리스가 움직이는 원리

게임판 배열

const ROWS = 20;
const COLS = 10;

const board = Array.from(
  { length: ROWS },
  () => Array(COLS).fill(0)
);

현재 블록 정보

const player = {
  matrix: null,
  x: 3,
  y: 0,
  score: 0,
  level: 1
};

회전회전한 모양이 벽에 닿는가?행과 열을 바꾸고 순서를 뒤집기
낙하아래 칸이 비어 있는가?비어 있으면 y 증가, 아니면 보드에 고정

게임 오버새 블록이 시작부터 충돌하는가?게임 중단 후 재시작 버튼 표시
4. 추천 파일 구조

처음에는 3개 파일로 나누기

├── style.css    # 디자인과 반응형 화면

연습 단계에서는 모든 코드를 하나의 index.html 파일에 넣어도 됩니다. 기능이 많아지면 HTML, CSS, JavaScript를 분리하는 편이 관리하기 쉽습니다.

5. 핵심 예제 코드

회전·충돌·줄 삭제의 기본 형태

블록 회전

function rotate(matrix) {
  return matrix[0].map((_, columnIndex) =>
    matrix.map(row => row[columnIndex]).reverse()
  );
}

충돌 판정

function collide(board, player) {
  const { matrix, x, y } = player;

  for (let row = 0; row < matrix.length; row++) {
    for (let col = 0; col < matrix[row].length; col++) {
      if (
        matrix[row][col] !== 0 &&
        (
          board[y + row] === undefined ||
          board[y + row][x + col] === undefined ||
          board[y + row][x + col] !== 0
        )
      ) {
        return true;
      }
    }
  }
  return false;
}

완성된 줄 삭제

function clearLines() {
  let lines = 0;

  for (let y = board.length - 1; y >= 0; y--) {
    if (board[y].every(cell => cell !== 0)) {
      board.splice(y, 1);
      board.unshift(Array(COLS).fill(0));
      lines++;
      y++;
    }
  }

  const scoreTable = [0, 100, 300, 500, 800];
  player.score += scoreTable[lines] * player.level;
}
6. 멋지게 보이게 만드는 법

게임의 품질은 시각·소리·반응에서 결정됩니다

네온 블록

각 블록에 밝은 색상, 내부 하이라이트, 바깥쪽 그림자를 넣습니다.

부드러운 애니메이션

줄 삭제 순간에 짧은 빛 효과와 흔들림 효과를 넣습니다.

사운드 피드백

이동, 회전, 고정, 줄 삭제, 게임 오버마다 다른 효과음을 사용합니다.

다음 블록 표시

플레이어가 미리 전략을 세울 수 있도록 다음 블록을 보여줍니다.

고스트 블록

현재 블록이 떨어질 위치를 반투명하게 표시합니다.

모바일 조작

화면 아래에 좌우·회전·낙하 버튼을 배치합니다.

7. 고급 발전 아이디어

기본 완성 후 추가할 기능

게임 기능

  • 홀드 블록 기능
  • 콤보 및 백투백 보너스
  • 레벨별 배경 변화
  • 최고 점수 저장
  • 일시정지 및 재시작
  • 난이도 선택

기술 발전

  • LocalStorage로 최고 점수 저장
  • Canvas API로 빠른 화면 렌더링
  • Web Audio API로 사운드 제어
  • GitHub Pages로 무료 공개
  • PWA로 스마트폰 설치 지원
  • 온라인 순위표 연결

다른 게임의 이름, 로고, 음악, 그래픽을 그대로 복제하지 말고 직접 만든 디자인과 사운드를 사용하는 것이 안전합니다.

8. 완성 체크리스트

아래 항목이 모두 되면 1차 완성입니다

  • 게임판이 10×20 크기로 표시된다.
  • 7종 블록이 무작위로 등장한다.
  • 좌우 이동과 회전이 정상 작동한다.
  • 블록이 바닥이나 다른 블록 위에 고정된다.
  • 완성된 줄이 삭제된다.
  • 점수와 레벨이 표시된다.
  • 블록이 맨 위까지 쌓이면 게임 오버가 된다.
  • 재시작 버튼으로 다시 시작할 수 있다.
  • 휴대폰 화면에서도 UI가 깨지지 않는다.
  • 효과음과 애니메이션이 과하지 않다.
